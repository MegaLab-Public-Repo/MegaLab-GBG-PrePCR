Agilent Bravo Driver (Agilent.Bravo2018.Driver)

Release Notes

Known Issues

  • [Issue] None.

Version 4.5.6 (December 21, 2020)

Fixes

  • [Fixed] Fixed issue causing timeout in Green Button Go when some VWorks protocols included shutdown protocol and others did not.

Version 4.5.5 (October 7, 2020)

Improvement

  • [Improvement] Updated documentation.

Version 4.5.4 (June 11, 2020)

New

  • [Feature] Added 'Set Parameter' command. This command automates the editing of VWorks protocol files.

Improved

  • [Improvement] Removed 'Get Variable' and 'Set Variable' commands and sequence steps from the instrument. Those commands would throw a 'This method is not implemented.' exception. Though they are core Liquid Handler commands, the VWorks API does not support passing variables.
  • [Improvement] Added extra documentation on registering VWorks.exe.

Version 4.5.3 (May 12, 2020)

Fixes

  • [Fixed] Implemented logic to handle situation where after handling an error through VWorks, the protocol completes while GBG still showing the error screen.

Improved

  • [Improvement] Added check for VWorks log. Error message will be shown in GBG if VWorks shows 'VWorks is now running in Reduced Speed Mode.' message. That message box shown in VWorks does not trigger the message box event and so GBG did not notify the user that VWorks was waiting on a user response.

Version 4.5.2 (April 8, 2020)

Fixes

  • [Fix] Fixed error message connecting to the Bravo if Green Button Go isn't running as Administrator.

Version 4.5.1 (January 9, 2020)

Fixes

  • [Fix] Added capturing of a Shutdown protocol starting so that the GBG will not see the protocol as complete after the Main protocol has finished but a Shutdown protocol has started. Note, a protocol with only a startup protocol and without a Main or Shutdown protocol will not properly report as complete.

Improved

  • [Improvement] Added Kill VWorks command to force VWorks to shutdown if needed.

Version 4.5 (November 19, 2019)

New

  • [New] Added option to subscribe to User Messages from VWorks. User Messages will be treated like errors to notify user that VWorks needs user interaction.

Improved

  • [Improvement] Moved all calls to VWorks API to separate thread to better catch VWorks errors.

Version 4.4 (November 14, 2019)

New

  • [New] Added Move Protocol sequence step and instrument command. Command relies on a VWorks protocol parsing a file to which GBG writes the source and destination nests.

Improved

  • [Improvement] Added code to automatically initialize VWorks if command is called before the Initialize command is called.
  • [Improvement] Driver now throws error if VWorks Message contains 'GetController failed for controllerID' to allow GBG to catch VWorks error that is not being properly passed to GBG.

Version 4.3 (October 23, 2019)

New

  • [New] Added Load Device File command.

Version 4.2

Improved

  • [Improvement] Removed close protocol call from end of run protocol.
  • [Improvement] Implemented Load Protocol and Run Protocol commands instead of notifying user to use Load and Run Protocol command.
  • [Improvement] Added Close Protocol command.

Version 4.1.0
