Biosero Liquid Handler Driver (Biosero.GreenButtonGo.LiquidHandler)

Release Notes

Version 1.6.2 (September 23, 2020)

Improved

  • [Improvement] Better implementation of error handling if the user selects 'Modify', 'Retry', or 'Ignore' on a 'Run Sequence Activities' command error instead of 'Continue'. On these selections, the step will always start from the beginning. If 'Ignore' is selected, the step will now increment automatically, ignoring the step that errored.

Version 1.6.1 (September 14, 2020)

Fixed

  • [Fix] Better implementation of caching variable value for Continue action for 'Run Sequence Activities' command.

Version 1.6.0 (June 16, 2020)

Fixed

  • [Fix] Fixed cloning of 'Set Current Step' sequence step.
  • [Fix] Fixed issue with output variables not being properly synced by 'Run Instrument Command' sequence step.
  • [Fix] No longer create a new instance of instrument commands during execution of 'Run Instrument Command' sequence steps.

Version 1.5.11 (April 20, 2020)

New

  • [Feature] Added 'Set Current Step' sequence step. This allows for easy setting current step back to previous step to create a loop, or forward to skip steps.

Version 1.5.10 (April 7, 2020)

Fixed

  • [Fix] Fixed variable value not being passed back to instrument command on Continue action for 'Run Sequence Activities' command.
  • [Fix] Fixed issue with cached value for for error recovery when running multiple Liquid Handlers.

Version 1.5.8 (February 21, 2020)

Improvement

  • [Improvement] Improved Get Plate and Put Plate procedures for Liquid Handlers that have extend nest and/or retract nest commands.
  • [Improvement] Added extra user warning if anything other than 'Continue' is selected from the error dialog on a Run Sequence Activities command. This gives the user a second chance to view the Sequence Error Correction window.

Version 1.5.6 (January 8, 2020)

Fixed

  • [Fix] Variables values will be passed into command properties for Run Instrument Commands instead of the variable names.
  • [Fix] Added "Sequence Name" to the variables automatically created by Run Sequence process.

Version 1.5.4 (November 27, 2019)

Fixed

  • [Fix] Fixed 'ProtocolName' variable for data publishing messages.
  • [Fix] Fixed cloning of 'Run Instrument Command' steps.
  • [Fix] Fixed data publishing to work with remote client.

Version 1.5.1 (November 14, 2019)

Improvement

  • [Improved] Renaming of instrument now also renames and modifies sequence file.
  • [Improved] Added better error message if driver wasn't initiated before calling instrument commands.

New

  • [New] Added data publishing.
  • [New] Added Lock and Unlock sequence steps for Liquid handler to demarcate critical steps and lock the execution of sequences so that only the current sequence can execute.

Version 1.5.0 (July 24, 2019)

Improvement

  • [Improved] Added validation to sequence name.
  • [Improved] Added HelpText property to sequences and display as ToolTips in sequence editor.

Version 1.4.0 (June 27, 2019)

Improvement

  • [Improved] Made the Continue error handling window non-modal so that on an error, other parts of the program can be modified.

Version 1.3.0 (June 14, 2019)

Fixed

  • [Fix] Fixed the issue with step counter error on quick stop/start of integration.
  • [Fix] Reset stop flag so that simulation times would work correctly after a stop/restart.
  • [Fix] Fixed Sequence Number ordering in Sequence Editor.
  • [Fix] Fixed the issue with cloned sequence shows 0 for all step numbers.

New

  • [New] Added the Run GBG Script Step.
  • [New] Added the Set GBG Variable Step.
  • [New] Added conditional option to Run Instrument Command step.
  • [New] Allowed sequence name to be specified with variable.

Version 1.2.0 (March 26, 2019)

New

  • [New] Added deck layout, sequence process base class, and sequence instrument base class to allow for easier creation of drivers using this base project.
